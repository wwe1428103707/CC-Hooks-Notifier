# 托盘通知中心 — 需求报告

> 版本: v1.0  
> 日期: 2026-05-24  
> 状态: 待评审

---

## 1. 需求概述

参照 QQ/微信的托盘体验，为 CC Hooks Notifier 增加**持续闪烁提示 + 未读计数 + 单击查看详情**的通知中心功能。用户无需主动打开面板即可感知有新通知，悬停可查看未读数量，单击直达详情页。

### 当前行为 vs 目标行为

| 维度 | 当前 | 目标 |
|------|------|------|
| 闪烁 | 定时器驱动，10 秒/5 秒后自动停止 | **持续闪烁**，直到用户单击托盘图标 |
| 悬停提示 | 静态文本 `Claude Code Hooks Notifier` | 动态显示未读数量，如 `3 条未读通知` |
| 单击 | 停止闪烁（不打开面板） | **打开主面板**，同时清除闪烁和未读 |
| 双击 | 打开主面板 | 保持不变（与单击行为一致） |
| 未读标记 | 无 | 事件分为已读/未读，面板中高亮未读项 |
| 图标角标 | 无（仅蓝/橙二色切换） | 可选项：动态生成带数字角标的图标 |

## 2. 功能需求

### 2.1 持续闪烁 (Persistent Blink)

**FR-1.1** 当有新通知到达时，托盘图标开始闪烁（蓝/橙交替，间隔 500ms）。

**FR-1.2** 闪烁**持续进行**，不设自动停止计时器。只有以下操作会停止闪烁：
- 用户单击托盘图标
- 用户双击托盘图标
- 用户通过托盘菜单点击"查看通知"

**FR-1.3** 如果用户在闪烁期间再次收到新通知，闪烁继续（不重置、不叠加）。

**FR-1.4** 多个通知同时到达（短时间内批量推送），闪烁只启动一次，不重复触发。

### 2.2 悬停提示未读计数 (Tooltip with Unread Count)

**FR-2.1** `NotifyIcon.Text`（tooltip）动态更新，格式基于当前语言：

| 语言 | 无未读 | 有未读 |
|------|--------|--------|
| 中文 | `Claude Code Hooks Notifier` | `3 条未读通知 — Claude Code Hooks Notifier` |
| English | `Claude Code Hooks Notifier` | `3 unread notifications — Claude Code Hooks Notifier` |

**FR-2.2** 未读计数在以下时机更新：
- 新通知到达（IPC 消息）→ 计数 +1 → 更新 tooltip
- 用户打开主面板 → 计数归零 → 更新 tooltip
- 用户在面板中手动"全部标为已读"→ 计数归零 → 更新 tooltip

**FR-2.3** 未读计数仅在**托盘进程**中维护（内存计数器），不依赖面板是否打开。

### 2.3 单击打开主页面 (Single-Click Opens Dashboard)

**FR-3.1** 托盘图标的**单击**行为改为：
1. 停止闪烁
2. 将所有未读通知标记为已读
3. 打开 `MainWindow` 面板（如果已打开则聚焦）
4. 面板自动切换到 **Event Log（事件日志）** 标签页，并高亮新通知

**FR-3.2** 双击行为与单击保持一致（都打开面板并清除未读）。

**FR-3.3** 托盘右键菜单新增一项：**"查看通知 (N 条未读)"**，点击行为与单击一致。

### 2.4 通知详情查看 (Notification Detail View)

**FR-4.1** 主面板 **Event Log** 标签页需要展示每条事件的已读/未读状态：
- 未读条目左侧显示**醒目圆点**（蓝色/橙色，对应 P0/P0.5 等级）
- 已读条目左侧无圆点，或灰色小点

**FR-4.2** 未读条目在列表中**置顶**（按时间倒序排列，但未读始终在已读上方）。

**FR-4.3** 新增"未读"筛选标签：全部 / 未读 / P0 / P0.5 / Toast，默认为"未读"（当从托盘单击进入时）。

**FR-4.4** 每条事件展开后显示完整详情（Timestamp, Level, Event Name, Summary, Detail）。

**FR-4.5** 面板右上角显示未读计数徽章，点击可一键"全部标为已读"。

### 2.5 已读/未读状态管理 (Read/Unread State)

**FR-5.1** `EventEntry` 数据模型新增 `bool IsRead` 字段（默认 `false`）。

**FR-5.2** 已读状态持久化到 `event_history.json`，重启后保留。

**FR-5.3** 标记已读的时机：
- 用户单击/双击托盘图标 → **全部**标记已读
- 用户在面板中点击"全部标为已读"按钮 → 全部标记已读
- 用户在面板中点击单条未读事件 → 该条标记已读
- 用户在面板中展开单条未读事件（detail row）→ 该条标记已读

**FR-5.4** 未读计数上限 999（避免 tooltip 过长），超过显示 `999+`。

### 2.6 图标角标数字（可选 / Phase 2）

**FR-6.1** 使用 GDI+ 动态生成带数字角标的托盘图标（类似 QQ 在任务栏上的未读计数）。

**FR-6.2** 数字绘制在图标右下角，白字红底/橙底圆形徽章。

**FR-6.3** 数字 > 99 时显示 `99+`。

**FR-6.4** 此功能为可选增强，Phase 1 可先用 tooltip 替代。

> **注意：** Windows 系统托盘图标为 16×16 像素，角标数字可读性较差。此功能建议作为 Phase 2 可选特性，优先通过 tooltip 和面板满足需求。

## 3. 数据模型变更

### 3.1 EventEntry（EventHistory.cs）

```csharp
// 当前
record EventEntry(DateTime Timestamp, string Level, string EventName, string Summary, string Detail = "");

// 目标
record EventEntry(
    DateTime Timestamp,
    string Level,
    string EventName,
    string Summary,
    string Detail = "",
    bool IsRead = false       // 新增
);
```

**兼容性说明：** `event_history.json` 中旧条目反序列化时 `IsRead` 默认为 `false`，视为未读。首次启动后用户单击托盘即可全部标为已读。

### 3.2 EventHistory 新增方法

```csharp
int UnreadCount { get; }                        // 未读事件总数
int UnreadCountByLevel(string level);            // 按等级统计未读
void MarkAllRead();                              // 全部标为已读
void MarkRead(int index);                        // 单条标为已读
List<EventEntry> GetUnread();                    // 获取所有未读事件
```

## 4. IPC 协议变更

### 4.1 无需变更

当前 IPC 消息格式已经包含了所有必要字段（`BlinkType`, `Title`, `Body`, `EventName`, `EventType` 等）。未读计数在托盘端自动维护，无需修改 Wire Format。

### 4.2 可选增强：通知计数查询

如果未来面板需要独立于托盘进程运行（如 WebSocket 模式），可新增 IPC 消息类型：

```json
{
  "Protocol": 1,
  "Type": "query_unread",
  "RequestBody": ""
}
// Response:
{ "status": "ok", "unreadCount": 3 }
```

Phase 1 不需要此增强——面板始终由 `MainWindow` 的 `WebView2` 承载，C# 端可以直接通过 `PushState` 推送未读计数。

## 7. 风险与注意事项

| 风险 | 等级 | 缓解措施 |
|------|------|---------|
| **已读状态持久化文件变大** | 低 | 当前上限 500 条，每条加 1 bool，影响可忽略 |
| **旧版 JSON 反序列化兼容** | 低 | `IsRead` 使用 `= false` 默认值，旧数据自动兼容 |
| **闪烁持续导致 CPU 占用** | 低 | 500ms 间隔的 WinForms Timer 资源消耗极低 |
| **大量未读时 tooltip 过长** | 低 | 设置 999 上限，超过显示 `999+` |
| **WebView2 面板未加载时状态同步** | 中 | 面板打开时通过 `state_sync` 一次性同步全量状态 |
| **GDI+ 角标图标在 16×16 下不可读** | 中 | Phase 2 可考虑仅 32×32 下绘制角标，16×16 使用无角标图标 |

---

## 8. 验收标准

1. 新通知到达 → 托盘图标开始闪烁，不自动停止
2. 鼠标悬停托盘图标 → tooltip 显示 `N 条未读通知`
3. 单击托盘图标 → 闪烁停止，面板打开并显示未读事件（高亮）
4. 关闭面板 → 未读计数归零，tooltip 恢复默认
5. 重启应用 → 未读状态从 `event_history.json` 恢复（之前未读的仍显示未读）
6. 面板筛选功能正常（全部/未读/P0/P0.5/Toast）
7. "全部标为已读"按钮功能正常
8. 中英文 i18n 完整覆盖新增文案
